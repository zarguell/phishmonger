LureList Test Scenarios

Test Case 1: Cross-element marking (3 spans, same UUID)

This should appear as ONE lure entry in LureList with combined text.

URGENT: Click here now! Action required.

Expected Result:

  • LureList shows: 1 entry (not 3)
  • Text preview: "URGENT: Click here now! Action required."
  • UUID: test-uuid-123 (truncated to 8 chars)

Test Case 2: Remove lure button

Click the × button next to a lure to remove it.

This is highlighted text that should be removed.

Expected Result:

  • Click × button next to lure
  • All spans with matching UUID are removed from HTML
  • Text content is preserved: "This is highlighted text that should be removed."
  • Preview re-renders without yellow highlights
  • LureList updates to show one less lure

Test Case 3: Multiple lures, different UUIDs

Multiple separate lures should appear as separate entries.

First lure and Second lure and Third lure

Expected Result:

  • LureList shows: 3 separate entries
  • Each has its own UUID
  • Each has its own remove button

How to Test in App:

  1. Open app at http://localhost:5176/
  2. Switch to "HTML Input" mode
  3. Paste one of the test cases above into the textarea
  4. Observe the LureList in the right sidebar
  5. Click the × button to test removal
  6. Check HTML source to verify spans are removed
